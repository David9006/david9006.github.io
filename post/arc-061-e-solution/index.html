ARC 061 E 题解

2024-02-16 OI

Link: https://atcoder.jp/contests/arc061/tasks/arc061_c

Wrong Solution

First of all, we should know that the whole map should be considered as a graph with stations as vertices and railways as edges. Now a solution should appear in your brain: run 0-1 BFS with an edge weighted:

  • 111 if the endpoints aren’t managed by the same company, and
  • 000 otherwise.

Unfortunately, it failed at 21 (out of 59) test cases.

Right Solution

While we store the graph (be sure to make it directed), we split the node uuu into multiple nodes: (u,ci)(u, c_i)(u,ci​) with cic_ici​ from all companies of every edges node uuu be connected to. For instance, we can split node 111 in Sample Input 1 into (1,1)(1,1)(1,1) and (1,2)(1,2)(1,2). Then we connect every pair of nodes (ui,ci)(u_i, c_i)(ui​,ci​) and (vi,ci)(v_i, c_i)(vi​,ci​) if, in the original graph, uiu_iui​ and viv_ivi​ have an edge weighted cic_ici​.

Now another problem appears: how do we do transfers? We can set up nodes like (ui,0)(u_i, 0)(ui​,0) and connect every (ui,ci)(u_i, c_i)(ui​,ci​) with it, with edges weighted 000, and connect (ui,0)(u_i, 0)(ui​,0) to every (ui,ci)(u_i, c_i)(ui​,ci​) with edges weighted 111.

At last, we only need to run Dijkstra on the new graph that begins at (1,0)(1, 0)(1,0) and ends at (n,0)(n, 0)(n,0) and the problem is solved.

By the way, be careful when writing the code!

Time complexity: O((N+M)log⁡M)O((N+M) \log M)O((N+M)logM)
